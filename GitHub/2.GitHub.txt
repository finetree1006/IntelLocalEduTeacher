2.GitHub.txt


1)깃허브란?
Git을 사용할 때 소스코드를 올려놓는 서버가 필요한데, 
이 서버를 제공해 주는 서비스

2)기능
.소스코드 관리
.프로젝트 설명서
.커뮤니티

.깃허브 이용비용
-오픈소스 프로젝트: 무료
-사용 private 프로젝트: 유료

*필요 이유
-개인적으로 서버를 구현하는 것은 매우 번거로움
-GitHub에 프로젝트만 생성하면 누구나 쉽게 Git의 중앙 서버로
Github를 사용할 수 있음
-각 개발자들의 소스코드의 상태를 관리.저장해 주는 저장소 역할
-안전하게 프로젝트를 진행할 수 있음
-다른 이들이 만든 프로젝트들을 쉽게 찾고 사용할 수 있음
-다른 이용자와 질문과 답변 가능
-통계 기능 사용 가능





*Repository
Add .gitignore: 파일 업로드 하기 전에 업로드를 무시할 파일 형태를 지정
- 순수 해당 프로젝트에서 관리할 소스코드들만 관리할 수 있도록 설정
- 개발자 각 환경에서 자동으로 생성되는 빌드 파일 등을 제외시키는데 사용
- gitignore를 사용하지 않고 모든 소스코드를 포함시킬 경우,
빌드 파일등 특정 환경에서만 변경되는 파일들이 자꾸 수정사항으로 반영되어
관리가 어려워 짐


*Repository 화면 구성
Master Branch: branch 표시
해당 저장소에서 사용되고 있는 branch 수. 브랜치가 여러 개 생성되면 값이 변경됨

issue/ pull request
해당 저장소에 현재 이슈와 pull request 수

contributor:
해당 저장소에 참여하는 참여자
기여자가 많다면 많이 활성화된 오픈소스 프로젝트라고 판단하는 척도가 됨

Settings
저장소의 기능 수정

SSH clone URL
해당 저장소를 사용자의 로컬 환경에서 URL을 통하여 내용을 가져올 수 있게 하는 주소 생성
